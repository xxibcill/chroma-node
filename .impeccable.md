## Design Context

### Users
Chroma Node is for developers and color-grading learners using a desktop app to inspect one clip, understand grading mechanics, and experiment with professional-style controls.

### Brand Personality
Precise, grounded, studio-like. The interface should feel calm, technically credible, and focused on image work instead of ornamental chrome.

### Aesthetic Direction
Dark desktop workstation UI with the viewer as the anchor. Space efficiency is the first priority. The app should feel like a compact grading console, not a dashboard with decorative shell chrome.

### Design Principles
- Keep the primary grading workflow inside a typical laptop viewport on desktop.
- Let the viewer stay dominant while supporting controls remain immediately reachable.
- Use internal panel scrolling for overflow instead of document scrolling.
- Favor compact, legible controls and horizontal grouping over unnecessary vertical growth.
- Treat top-level chrome as expendable. If a header, border, or card does not improve the workflow, remove it.
- Prefer tight padding, short labels, and smaller workstation typography over roomy presentation styling.
- Use borders sparingly and only to clarify interaction or grouping.
- Add regression coverage when changing shell layout so viewport-fit behavior stays enforced.
