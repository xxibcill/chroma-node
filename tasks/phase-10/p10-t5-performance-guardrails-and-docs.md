## Scope

- Add messages or indicators when the app falls back to proxy behavior.
- Document supported and best-effort performance envelopes.
- Update verification and release notes with large-media guidance.

## Implementation Notes

- Guardrails should prevent silent degradation where the user might trust an inaccurate preview path.
- Documentation should separate functional support from recommended working ranges.

## Acceptance Criteria

- The app communicates when large-media fallbacks are active.
- Documentation describes supported and best-effort raster ranges.
- Verification guidance includes large-media checks and expected fallback behavior.
