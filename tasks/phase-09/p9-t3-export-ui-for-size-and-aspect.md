# [P9-T3] Export UI For Size And Aspect

## Status

Not started

## Phase

[Phase 09 - Flexible Export Geometry](../../roadmap/phase-09-flexible-export-geometry.md)

## Outcome

Expose output size, aspect, and resize policy controls in the export workflow.

## Scope

- Add preset choices for common social and standard delivery sizes.
- Allow custom width and height entry.
- Show the final export plan clearly before encode starts.

## Implementation Notes

- The UI should state when a preset changes aspect ratio versus only changing size.
- Final export labels should help the user understand whether the app will fit, crop, or pad.

## Acceptance Criteria

- The user can choose a preset or enter custom dimensions from the export UI.
- The selected resize policy is visible and understandable.
- The UI surfaces the final planned output raster before export begins.
