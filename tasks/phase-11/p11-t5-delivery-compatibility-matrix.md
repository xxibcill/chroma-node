## Outcome

Document and verify the expanded delivery surface across formats, geometry, and audio behavior.

## Scope

- Define a verification matrix for codec, container, geometry, and audio combinations.
- Add focused automated or scripted checks where practical.
- Update release notes and roadmap status with delivery compatibility limits.

## Implementation Notes

- Verification should emphasize user-visible compatibility, not only FFmpeg command success.
- Keep the matrix small enough to run regularly but broad enough to catch cross-feature regressions.

## Acceptance Criteria

- A documented delivery compatibility matrix exists.
- Verification covers the supported delivery combinations that ship.
- Release-facing documentation reflects actual supported and unsupported delivery paths.
